# Incident Engineering Starter Kit

A collection of practical resources for engineers investigating production incidents.

---

# Contents

## 1️⃣ Production Incident Starter Kit  
📄 [Download PDF](./pdf/Production%20Incident%20Starter%20Kit%20v1.pdf)

A practical starter bundle designed to help engineers investigate production incidents more effectively.

The kit includes a **debugging cheat sheet, example AWS log queries, and a postmortem template** that teams can use immediately during incident investigations. It provides a simple starting framework for **DevOps engineers, SREs, and backend developers** dealing with real production failures.

---

## 2️⃣ Incident Response Playbook  
📄 [Download PDF](./pdf/Incident%20Response%20Playbook%20v1.pdf)

A structured guide explaining how experienced engineering teams handle production incidents under pressure.

The playbook walks through the **five phases of incident response**:

- Detection  
- Stabilisation  
- Investigation  
- Resolution  
- Learning  

It provides practical techniques for coordinating response efforts, assigning roles, and restoring system stability during outages.

---

## 3️⃣ 20 Production Failure Patterns  
📄 [Download PDF](./pdf/20%20Production%20Failure%20Patterns%20v1.pdf)

A field guide to the most common failure scenarios engineers encounter in modern distributed systems.

This resource helps engineers quickly recognise patterns such as:

- dependency timeouts  
- connection pool exhaustion  
- deployment regressions  
- resource saturation  
- cascading failures  

Recognising these patterns enables **faster root cause identification during production incidents**.

---

## 4️⃣ 10-Minute Debugging Framework  
📄 [Download PDF](./pdf/10-Minute%20Debugging%20Framework%20v1.pdf)

A simple investigation method engineers can use during the **first critical minutes of a production incident**.

The framework provides a structured triage process that helps responders quickly determine:

- whether the signal is real  
- the blast radius of the issue  
- recent system changes  
- shared dependencies  
- the most likely root cause hypothesis  

---

## 5️⃣ Ultimate Incident Engineering Toolkit  
📄 [Download PDF](./pdf/Ultimate%20Incident%20Engineering%20Toolkit%20v1.pdf)

A curated overview of essential tools used by modern **SRE and DevOps teams** to detect, investigate, and resolve production incidents.

The toolkit introduces engineers to key platforms across:

- observability  
- log analysis  
- tracing  
- incident management  
- chaos engineering  

Including tools such as:

- Grafana  
- Prometheus  
- Splunk  
- CloudWatch  
- PagerDuty  
- Jaeger
